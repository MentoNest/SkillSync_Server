## Summary

- Notification entity with read/unread tracking persisted in PostgreSQL
- `GET /notifications` with pagination and unread count; `POST /notifications/read` for single or batch mark-read
- Real-time delivery via Socket.io `/notifications` namespace (JWT-authenticated)
- Rate limited to 100 notifications/hour per user via Redis
- `DELETE /notifications/cleanup` purges notifications older than 90 days
